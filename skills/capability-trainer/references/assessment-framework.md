# Self-Assessment Framework

Simplified assessment for capability learning.

## Practical Mastery Checklist

After completing practice tasks, assess:

```
Can I:
□ Explain what this does in simple terms?
□ Perform common tasks without looking up docs?
□ Recognize when to use this tool/technique?
□ Debug basic issues on my own?
□ Apply to a slightly different scenario?
```

**If unchecked items**:
- Identify specific gap
- Practice that area
- Reassess

## Expert Level Assessment

More thorough check for expert-level mastery:

```
Can I:
□ Explain core concepts to a beginner?
□ Work independently without any reference?
□ Handle edge cases and unusual situations?
□ Debug complex issues systematically?
□ Apply to completely new scenarios?
□ Choose right approach among alternatives?
□ Identify and avoid common pitfalls?
```

## Validation Methods

### Method 1: Task Verification
Design a task you haven't practiced. Complete it without references.
- Success = Can do it
- Partial success = Needs more practice
- Failed = Gap identified

### Method 2: Explanation Test
Explain a concept in simple terms as if teaching someone new.
- Clear = Good understanding
- Confusing = Needs review

### Method 3: Edge Case Testing
Try unusual inputs or scenarios.
- Handles correctly = Strong mastery
- Breaks or unsure = Needs more practice

### Method 4: Comparison
Compare your work with official examples or best practices.
- Match or better = Excellent
- Clear gaps = Areas to improve

## When to Stop Practicing

**Practical level**: When you can complete common tasks independently

**Expert level**: When you can handle unfamiliar scenarios and explain concepts clearly

**Don't aim for perfection**. Aim for "good enough to work independently, know where to look when stuck."

## Example: Git Workflow Assessment

**Practical level check**:
- [ ] Can create branch and commit changes
- [ ] Can merge without conflicts
- [ ] Can resolve simple conflicts
- [ ] Can check history and undo mistakes

**Expert level check**:
- [ ] Can explain when to use merge vs rebase
- [ ] Can resolve complex conflicts
- [ ] Can recover from lost commits
- [ ] Can choose appropriate workflow for team
